# Project guidance for Claude

## Always open a pull request for code changes

Whenever you generate or modify code in this repo, open a GitHub pull request for
that work — every time, without waiting to be asked.

- This applies to any code change: features, fixes, refactors, config, hooks,
  scripts, etc.
- Commit and push to the working branch, then open the PR against the default
  branch (`main`).
- Do this proactively as the default. The user would rather decline or close an
  unwanted PR in a specific chat than have changes left without one.
- If the user explicitly says they don't want a PR in a given conversation,
  honor that for that conversation only — it does not change this standing rule.
- Documentation-only or memory-file edits (like this file) still count as
  changes worth a PR.
